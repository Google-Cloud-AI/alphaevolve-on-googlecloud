# Visualization Reference

Guide for generating visual artifacts from experiment results using the
`ae` CLI's built-in visualization commands.

--------------------------------------------------------------------------------

## 1. Score Progression Chart (PNG)

Generate a combined scatter + running-best line chart as a PNG image.

### Command

```bash
ae results plot <EXPERIMENT> --output <PATH> [--title <TITLE>]
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--output`, `-o` | `score_progression.png` | Output PNG file path |
| `--title` | `Experiment: <nickname>` | Chart title |

### Examples

```bash
# Basic usage
ae results plot brave-otter

# Custom output path and title
ae results plot brave-otter \
  --output /home/user/my_experiment/chart.png \
  --title "Circle Packing Optimization"
```

### What it produces

The chart shows all evaluations as a scatter plot with the running best
score overlaid as a line:

| Element | Visual | Purpose |
|---------|--------|---------|
| Blue scatter dots | `#6495ED`, alpha=0.6 | All successful evaluations |
| Green step line | `#2ecc71`, linewidth=2.5 | Running best score |
| Green star | `#2ecc71`, size=120 | Best program (annotated with nickname + score) |
| Red dashed line | `#e74c3c` | Baseline score |
| Grey X markers | `#cccccc` | Failed evaluations |

### When to use

- **Post-experiment (Stage 1):** Always generate after gathering results.
  Reference the PNG in the markdown report:
  ```markdown
  ![Score Progression](score_progression.png)
  ```
- **During monitoring:** Generate periodically to update the dashboard
  with a visual chart (see Section 3).

## 2. Interactive HTML Report

Generate a self-contained HTML file with an interactive chart and the
full experiment report embedded below it.

### Command

```bash
ae results report <EXPERIMENT> --output <PATH> [--markdown <MD_PATH>]
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--output`, `-o` | `experiment_report.html` | Output HTML file path |
| `--markdown` | (none) | Path to a markdown report file to embed below the chart |

### Examples

```bash
# Basic usage (chart + leaderboard only)
ae results report brave-otter

# With embedded markdown report
ae results report brave-otter \
  --output /home/user/my_experiment/report.html \
  --markdown /home/user/my_experiment/experiment_report.md
```

### What it produces

A single self-contained HTML file (no external dependencies) with:

1. **Result banner** at the top: `baseline → best (+improvement%)`
2. **Interactive scatter+line chart** (same design as the PNG):
   - **Hover over any point** to see program nickname, score, and a
     code preview (first 30 lines of evolved code)
   - Green step line for running best, red dashed baseline
   - Failed evaluations shown as grey X markers
3. **Experiment report** rendered below the chart (from the `--markdown`
   file, or just the header metadata if no markdown is provided)

### How the HTML report uses the markdown file

The `--markdown` flag accepts the path to the `experiment_report.md`
that the agent generates in Step 3.5. The CLI reads this file and
embeds its content below the interactive chart. This means the agent
should:

1. Generate `experiment_report.md` first (Step 3.5)
2. Then generate the HTML report referencing it (Step 3.5b)

```bash
ae results report brave-otter \
  --markdown experiment_report.md \
  --output experiment_report.html
```

## 3. Monitor Dashboard Integration

Use the plot command during monitoring to update the dashboard with a
visual chart alongside the text-based dashboard.

### Workflow

After each progress poll that reveals new evaluations:

```bash
ae results plot <EXPERIMENT> --output score_progression.png
```

The dashboard markdown (generated by `--dashboard` flag on
`ae experiment run`) can reference the PNG:

```markdown
![Score Progression](score_progression.png)
```

The `--dashboard` flag on `ae experiment run` already generates a
text-based markdown dashboard. The plot image supplements it with a
high-quality visual chart.
